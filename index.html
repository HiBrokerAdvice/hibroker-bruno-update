05 / CORRECTION

Upload photo de profil.

L'upload de la photo de profil ne marchait pas correctement. Le fichier partait bien, mais il n'était pas stocké au bon endroit dans Supabase Storage. Du coup, la photo n'apparaissait nulle part.

AVANT

Chemin de stockage mal construit. Fichier uploadé mais introuvable.

APRÈS

Chemin corrigé dans Supabase Storage. L'image s'affiche partout : onboarding, dashboard, paramètres.

06 / CORRECTION

Upload logo dans les paramètres.

Le courtier pouvait uploader son logo pendant l'onboarding, mais impossible de le changer après dans les paramètres. Les policies de Supabase Storage bloquaient la mise à jour depuis cette page.

Corrigé : policies de storage mises à jour + chemin de fichier unifié entre onboarding et paramètres.

08 / AUTH & CONNEXIONS

OAuth Google & Microsoft.

Le courtier connecte son compte Google ou Microsoft en un clic. Les tokens sont chiffrés en AES-256-GCM et rafraîchis automatiquement par un cron.

Microsoft 365

Login OAuth → Graph API pour l'envoi email depuis la boîte Outlook du courtier.

Google / Gmail

Login OAuth → Gmail API pour l'envoi email + Google Places pour la fiche.

Sécurité : tokens chiffrés AES-256-GCM en base, refresh automatique toutes les nuits, rate limiting sur les endpoints OAuth.
09 / CORRECTION

Boucles de redirection.

Certains courtiers se retrouvaient coincés dans une boucle infinie entre l'onboarding et le dashboard. Le middleware n'avait aucun moyen de savoir si l'onboarding était terminé ou pas.

Corrigé : ajout d'un flag onboarding_completed en base. Plus de boucle.
10 / CORRECTION

Routage email corrigé.

Le système d'envoi avait un souci : quand le token du courtier avait expiré, l'email échouait sans prévenir personne. Maintenant, si un provider échoue, le système bascule automatiquement sur le suivant.

1
Microsoft 365 — essayer via Graph API
2
Gmail — si M365 échoue, bascule sur Gmail API

Tout est loggé pour savoir quel provider a été utilisé.

11 / CORRECTION

Analyse IA plus robuste.

L'analyse IA plantait parfois — timeout, dépassement de quota — et l'avis restait bloqué sans analyse. C'est réglé :

☑ Retry automatique (jusqu'à 3 tentatives)
☑ Bouton "Relancer l'analyse" pour le courtier
☑ Log des erreurs pour debug
